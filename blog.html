Discuss the scope of var, let, and const

'var', 'let', and 'const' are keywords in JavaScript used to declare variables. They have different scopes and behaviors, which can affect how you work with data in your code. Here's a breakdown of their scopes and usage:

'var'

Function Scope: Variables declared with var are function-scoped, meaning they are limited to the function in which they are declared. If declared outside of any function, they become globally scoped.

Hosting: Variables declared with var are hoisted to the top of their containing function or global context. This means you can use a var variable before it's declared in the code, but it will be initialized with undefined.

Reassignment: You can reassign values to var variables.

Example: 

                    
                        function exampleFunction() {
                            var x = 10;
                            if (true) {
                              var y = 20;
                            }
                            console.log(x); // 10
                            console.log(y); // 20 (not block-scoped)
                          }

'let'

Block Scope: Variables declared with let are block-scoped, which means they are limited to the block (enclosed by curly braces) in which they are declared, such as loops or if statements.

Hoisting: Like var, let variables are also hoisted to the top of their block, but they are not initialized, so you can't access them before declaration.

Reassignment: You can reassign values to let variables.

Example: 

                    
                        function exampleFunction() {
                            let x = 10;
                            if (true) {
                              let y = 20;
                              console.log(x); // 10
                            }
                            console.log(y); // ReferenceError: y is not defined
                          }

'const'

Block Scope: Variables declared with const are also block-scoped, just like let.

Hoisting: const variables are hoisted but not initialized, so you can't access them before declaration.

Immutability: Variables declared with const cannot be reassigned once they are assigned a value. However, if they are assigned an object or array, you can still modify the properties or elements of that object or array.

Example: 

                    
                        function exampleFunction() {
                            const x = 10;
                            if (true) {
                              const y = 20;
                              console.log(x); // 10
                            }
                            console.log(y); // ReferenceError: y is not defined
                          }

In modern JavaScript development, it's generally recommended to use let and const over var because they provide more predictable scoping behavior and help prevent bugs related to variable hoisting and unintended reassignments. const is particularly useful when you want to declare values that should not be changed after their initial assignment.
